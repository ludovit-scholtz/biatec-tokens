# Compliance Reporting Workspace

**Route**: `/compliance/reporting`  
**File**: `src/views/ComplianceReportingWorkspace.vue`  
**Auth required**: Yes

---

## Overview

The Compliance Reporting Workspace is an enterprise-grade view that aggregates compliance evidence from all configured sources into a single, regulator-ready summary. Operators, compliance managers, and auditors use this workspace to review launch-level compliance status and export evidence bundles for sign-off, procurement review, or regulatory submission.

---

## How It Supports Enterprise Sign-off

| Need | Feature |
|------|---------|
| Overall readiness at a glance | Overall Compliance Status banner with score meter |
| Critical blocker identification | Blockers list with icon + text labels (not color-only) |
| Regulatory-ready documentation | JSON and plain-text export |
| Stale evidence detection | Freshness indicators on each subsection |
| Audit trail | Links back to source workflows |
| Procurement-ready format | "What to do next" guidance when evidence is incomplete |

---

## Evidence Sources

Data is loaded from these localStorage keys on mount:

| Key | Purpose |
|-----|---------|
| `biatec_compliance_setup` | Jurisdiction configuration, KYC/AML requirements, investor eligibility |
| `biatec_whitelist_setup` | Whitelist posture, approved/pending investor counts |
| `biatec_kyc_aml_setup` | KYC/AML provider status, pending review count |
| `biatec_strict_signoff` | Backend strict sign-off evidence (surfaced via Evidence Pack) |
| `biatec_team_approvals` | Team approval state (surfaced via Evidence Pack) |

## Sections

### 1. Overall Compliance Status
- Overall `EvidenceStatus` derived from the worst status among all subsections
- Readiness score (0–100%) calculated from subsection completeness
- Active blockers listed with icons and text descriptions
- "What to do next" guidance for non-ready states

### 2. Jurisdiction Coverage
- Lists configured permitted and restricted jurisdictions
- Shows permitted and restricted counts
- Freshness indicator if data is >24 hours old
- Link to `/compliance/setup` for updates

### 3. Investor Eligibility
- Accredited investor requirement flag
- Retail investor permission flag
- Eligibility category tags
- Freshness indicator

### 4. KYC / AML Review Status
- KYC and AML requirement flags
- Provider configuration status
- Pending review count with warning
- Freshness indicator

### 5. Whitelist Posture
- Whitelist required/not-required flag
- Approved and pending investor counts
- Active whitelist ID
- Freshness indicator
- Link to `/compliance/policy`

### 6. Evidence Summary
- Table of all evidence sections with status and grade
- Link to the full Compliance Evidence Pack (`/compliance/evidence`)

### 7. Export Compliance Report
- **Export JSON**: Downloads a `ComplianceReportBundle` as `.json`
- **Export Text Report**: Downloads a plain-text structured report as `.txt`
- **Copy to Clipboard**: Copies the text report to the system clipboard
- Print hint for browser print dialog

---

## Export Formats

### JSON Export (`compliance-report-<timestamp>.json`)
A machine-readable `ComplianceReportBundle` object including:
- `generatedAt`, `launchName`, `overallStatus`, `readinessScore`
- `jurisdiction`, `kycAml`, `whitelist`, `investorEligibility` summaries
- `evidenceSections` array
- `exportVersion: "1.0"`

### Text Export (`compliance-report-<timestamp>.txt`)
A human-readable plain-text document with:
- Header: `COMPLIANCE REPORTING WORKSPACE — <UTC date>`
- Sections separated by `─` rule lines
- Compliance status, score, and per-domain details
- Footer: `This document was generated by Biatec Tokens Compliance Reporting Workspace.`

### Clipboard Copy
Same content as the text export, written to the system clipboard for pasting into emails, ticketing systems, or documentation.

## Freshness / Staleness Detection

Each subsection shows a `⚠ Stale evidence — last updated X hours/days ago` warning when its source data is older than **24 hours**. This prompts operators to re-run compliance checks before finalising an export.

## Accessibility Features

- **Skip link**: `<a href="#reporting-main">Skip to main content</a>` (WCAG 2.4.1)
- **Landmark roles**: `role="region"` with `aria-label` on main container; `role="progressbar"` on score bar
- **`aria-labelledby`**: Every section's `<h2>` id is referenced by its parent `<section aria-labelledby="...">`
- **Non-color-only status**: All status indicators combine an icon + text label (e.g. ✓ Ready, ⚠ Review Needed) — never color alone
- **Keyboard navigable**: All interactive elements (buttons, links) are reachable and operable via Tab / Enter / Space
- **`aria-live` on loading state**: Screen readers announce when data finishes loading
